1. Introducción

Propósito

Este documento describe la arquitectura de software del sistema de Programación de Distribución de Combustibles para YPFB-Refinación.

El objetivo principal del proyecto es centralizar en una Base de Datos toda la información y las operaciones que se realizan al momento de elaborar la Programación de Distribución de Combustibles, reemplazando los procesos manuales actuales basados en planillas Excel.

La aplicación será desarrollada con tecnologías actuales y ampliamente demandadas en el mercado de desarrollo de software, garantizando escalabilidad, mantenibilidad y seguridad.

YPFB-Refinación actualmente distribuye a sus clientes potenciales los siguientes productos:

Gasolina Gasolina Especial Gas Solvente Aguarras Kerosene Diésel Butano GBC Gasolina Premium Plus

Alcance

Módulo Administrativo

  • Gestión de productos: Gasolina, Gasolina Especial, Gas, Solvente, Aguarras, Kerosene, Diésel, Butano
  • Gestión de plazas/plantas (Clientes)
  • Gestión de Rutas
  • Gestión del tipo de Transporte

Módulo de Programación

  • Programación mensual: Programar la distribución de los productos de YPFB-Refineria a las plazas o plantas (Clientes) con capacidad de ajustes en tiempo real.
  • Programación diaria y Ajustes en tiempo real: Programar la distribución de los productos diario y ajustes en tiempo real.
  • Visualización de rutas en mapas interactivos
  • Sistema de notificaciones para cambios en las planificaciones: Se debe enviar un correo indicando los ajustes realizados, tanto a YPFB-Refineria y plantas
  • Dashboard interactivo de Programación mensual y con filtros
  • Generación de reportes

Objetivos del Sistema

Automatización

  • Reemplazar procesos manuales en Excel
  • Documentación y datos Centralizados
  • Reducción de tiempos de procesamiento
  • Envió de correos electrónicos Automatizados

Optimización

  • Gestión eficiente de rutas
  • Reducción de costos operativos

Visualización

  • Dashboard interactivo
  • Mapas de rutas en tiempo real
  • Reportes

2. Procesos Actuales

Situación Actual

El proceso actual de planificación y distribución de combustibles se realiza mediante métodos manuales que presentan varias limitaciones:

Problemas Identificados

  • Coordinación manual mediante correo electrónico
  • Distribución manual mediante registros en Excel
  • Demora de 3 días hábiles en el armado de Excel
  • Planificación en plantillas no amigables
  • Falta de visibilidad en tiempo real

Consecuencias

  • Ineficiencia en la planificación
  • Errores en la distribución
  • Falta de actualizaciones en tiempo real
  • Dificultad para realizar ajustes
  • Pérdida de tiempo en procesos manuales

Flujo Actual

  1. Reunión mensual para coordinar montos de distribución por cliente
  2. Creacion de la planilla Excel "Programacion PRODE Comercial"
  3. Análisis manual de rutas considerando transporte, costos, distancia y tiempo
  4. Planilla final con fechas y cantidades por producto
  5. Modificaciones durante el mes que no se reflejan en la planilla original

3. Requisitos del Sistema

Requisitos Funcionales

Módulo Administrativo

  • Gestión de productos: AV Gas, Jet Fuel, Solvente 3 Aguarras, Kerosene, Diésel Oil, Butano, Gas Oil, GLP, GLP Importado
  • Gestión de plazas/plantas (Clientes)
  • Gestión de Rutas
  • Gestión del tipo de Transporte

Módulo de Programación

  • Programación mensual de distribución
  • Programación diaria y ajustes en tiempo real
  • Visualización de rutas en mapas interactivos
  • Sistema de notificaciones para cambios en las planificaciones
  • Dashboard interactivo de Programación mensual con filtros
  • Generación de reportes

Requisitos No Funcionales

Rendimiento y Disponibilidad

  • Medir los tiempos de respuesta
  • Disponibilidad las 24 horas o parametrización de horas activas
  • Almacenamiento en base de datos con tablas históricas y limpieza de tablas principales cada cierto tiempo, lo sugerido es 6 meses, el tiempo tiene que ser parametrizable.

Seguridad

  • Autenticación con Google o correo electrónico
  • Seguridad JWT entre Frontend y Backend
  • Usuario con roles específicos y schema propio en base de datos

4. Tecnologías Utilizadas

Tecnologías Principales

Frontend: Blazor WebAssembly

Framework moderno para aplicaciones web interactivas con C#.

Backend: ASP.NET Core 8 Web API

Framework para construir APIs RESTful robustas y escalables.

Base de datos: SQL Server

Sistema de gestión de bases de datos relacional.

Tecnologías Complementarias

Tailwind CSS

Framework de CSS para diseño moderno y responsive.

Leaflet

Librería JavaScript para mapas interactivos.

JWT

JSON Web Tokens para autenticación segura.
